0. Introduction.

Some recursive functions are slow because they compute the same intermediate results many times. These functions may be implemented more efficiently by using memoization, in which each intermediate result is stored in a mutable table as it is computed. Whenever the function needs to compute an intermediate result, it checks the table to see if it has computed that result before. If it has, then it uses the result in the table, without computing it again.

2. Implementation.

In the lectures, you saw how memoization can use an array or an association list to record intermediate results so they need not be computed again. For this laboratory assignment, you must use a hash table instead. The OCaml library has a module called Hashtbl (inexplicably not called Hashtable) with functions for mutable hash tables. Here are brief descriptions of functions from Hashtbl you will need.

Hashtbl.create ~random: false  s

Create a new empty hash table and return it. Its size s is an integer greater than 0. It’s not obvious how to choose s, or even what it measures, but 1000 seems to work well. Some OCaml functions have optional name parameters, although this hasn’t been discussed in the lectures. Here Hashtbl.create is called with the name parameter ~random, whose value is false. The hash table will not work without this name parameter.

Hashtbl.add  t  k  v

In the hash table t, associate the key k with the value v. If k already has a value, then the new value v replaces it.

Hashtbl.find_opt  t  k

Try to find the value associated with the key k in a hash table t. Return the result as an instance of the type option, which is predefined like this.

type 'a option = None | Some of 'a ;;

If the key k has no corresponding value in t, then return None. If k has a value v in t, then return Some v. You can use pattern matching to find out which one is returned.

You must write at least the following OCaml functions. You may also write helper functions if you need them.

c n k

Compute C(n, k) using the recursion shown above, but do not use memoization.

memyC n k

A memoized version of c. It must compute the same results as c, but it must use a hash table to prevent computing the same intermediate result more than once. The hash table must not be visible to functions other than memyC. Hint: do not call c from memyC.

The execution time of memoized functions is important for this assignment. The function time tests how long a function takes to compute its value. It takes a string title and a function without arguments func as its parameters.

let time title func = 
  let t0 = Sys.time () 
  in let result = func () 
     in let t1 = Sys.time () 
        in Printf.printf "%s %f seconds\n" title (t1 -. t0) ; 
           result ;;

For example, the call (time "c test" (fun () -> c 8 4)) prints a message containing "c test" about how long c 8 4 takes to execute. It then returns the value of c 8 4.
