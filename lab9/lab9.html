0. Introduction.

In this laboratory assignment, you’ll translate three Lisp functions into equivalent OCaml functions. This will help explain how Lisp works, allegedly helping to write an interpreter for it later in the course. It will also show how Lisp and OCaml differ.

1. Theory.

In the lectures, we defined an OCaml type thing that represents a Lisp object. For this assignment, we’ll use a simpler version of thing, shown below. We’ve removed the parts about closures, environments, and primitives, because they’re not needed here.

type 
  thing = 
    Cons of thing ∗ thing | 
    Nil | 
    Number of int | 
    Symbol of string ;;

We’ll also use Lisp expressions that are briefly defined in the following table. The lectures described them in more detail. Remember that ‘‘Lisp’’ is really a name for a family of related languages. A real Lisp system might have functions other than these, or may define them differently.

 LISP 

 DESCRIPTION 

 nil

 Empty list, Boolean false. 

 t 

 Boolean true. 

 (and e₁ e₂ ... eₙ)

 (car e)

 (cdr e) 

 A list like e but without its first element.

2. Implementation.

You must translate three Lisp functions, every, substitute, and questy-equal, into equivalent OCaml functions. Your OCaml functions may need to raise exceptions if they are called with arguments different from what they expect. You may define additional helper functions if you wish.

2.1. Every.

A predicate is a function that tests its arguments, returning true if the test succeeds, and false if the test fails. The higher-order Lisp function every tests if all elements of the list elements make predicate succeed. (Actually it tests if no element makes predicate fail.)

(define every  
  (λ (predicate elements)  
    (if  
      (is-cons elements)  
      (if  
        (predicate (car elements))  
        (every predicate (cdr elements))  
        nil)  
      t)))

Here are some examples of every, where the symbol ‘⇒’ means returns.

(every (λ (n) (= n 0)) nil)

  ⇒  

t

(every (λ (n) (= n 0)) (quote (0)))

  ⇒  

t

(every (λ (n) (= n 0)) (quote (0 0)))

  ⇒  

t

(every (λ (n) (= n 0)) (quote (0 1)))

  ⇒  

nil

(every is-cons nil)

  ⇒  

t

(every is-cons (quote ((A) (B))))

  ⇒  

t

(every is-cons (quote (A (B))))

  ⇒  

nil

Your OCaml function every must have the type (thing -> bool) -> thing -> bool. Note that it must return an OCaml Boolean, not a Lisp Boolean.

2.2. Substitute.

The function substitute takes three arguments: a list called elements, a symbol called old, and an object called new. It returns a copy of elements in which all appearances of old are replaced by new.

(define substitute 
  (λ (elements old new) 
    (if 
      (is-cons elements) 
      (cons 
        (if 
          (= (car elements) old) 
          new 
          (car elements)) 
        (substitute (cdr elements) old new)) 
      nil)))

Here are some examples of substitute.

(substitute nil (quote A) (quote B))

  ⇒  

nil

(substitute (quote (A B A)) (quote A) (quote B))

  ⇒  

(quote (B B B))

(substitute (quote (X Y Z)) (quote A) (quote B))

  ⇒  

(quote (X Y Z))

(substitute (quote (A (B))) (quote B) (quote A))

  ⇒  

(quote (A (B)))

Your OCaml function substitute must have the type thing -> thing -> thing -> thing.

2.3. Questy-equal.

The Lisp function questy-equal tests if two Lisp objects left and right are equal. However, whenever the symbol ? appears in left, it acts like a ‘‘wild card’’ that is equal to anything.

(define questy-equal 
  (λ (left right) 
    (or 
      (= left (quote ?)) 
      (and 
        (is-cons left) 
        (is-cons right) 
        (questy-equal (car left) (car right)) 
        (questy-equal (cdr left) (cdr right))) 
      (= left right))))

Here are some examples of questy-equal.

     

(questy-equal (quote 1) (quote 1))

  ⇒  

t

(questy-equal (quote A) (quote 1))

  ⇒  

nil

(questy-equal (quote A) (quote A))

  ⇒  

t

(questy-equal (quote ?) (quote X))

  ⇒  

t

(questy-equal (quote (A B C)) (quote (A X Y)))

  ⇒  

nil

(questy-equal (quote (A ? ?)) (quote (A X Y)))

  ⇒  

t

(questy-equal (quote (A (B 1) (C 2 3))) (quote (A (B 1) (C 2 3))))

  ⇒  

t

(questy-equal (quote (A ? (C 2 ?))) (quote (A (B 1) (C 2 3))))

  ⇒  

t

Your OCaml function questyEqual must have the type thing -> thing -> bool. Again, it returns an OCaml Boolean, not a Lisp Boolean.
