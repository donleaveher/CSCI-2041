You will have the entire class period (fifty minutes) to complete the examination. Attend only one examination. Please attend the one for your lecture section if possible: there is no advantage to attending one examination over the other.
      The midterm examination will be ‘‘open notes,’’ so you will be allowed to use any amount of anything represented on paper to help you answer questions, including: class notes, laboratory assignments, project assignments, program listings, Xeroxed pages, etc. However, you will not be allowed to use electronic devices or published books. Please see the syllabus for details.
      The examination will cover everything discussed in class up to and including February 20, 2026. The following are possible topics for questions. The topics do not appear in any particular order. Not all these topics will necessarily appear on the examination.

  • What functional programming (applicative programming) is.

  • How functional programming differs from imperative programming.

  • How to translate a function from imperative to functional form.

  • How to translate a function from functional to imperative form.

  • What persistence means.

  • How to apply OCaml functions.

  • How partial application of functions works (‘‘curried functions’’).

  • How to write recursions.

  • How to write tail recursions.

  • How to tell if a function is tail-recursive.

  • How to translate a function to tail-recursive form (if possible).

  • How to write ‘‘helper’’ functions.

  • How to write ‘‘internal’’ and ‘‘external’’ helper functions.

  • How let, let–rec, let–in and let–rec–in work.

  • How if–then–else works.

  • How match–with works.

  • How p –> e rules work within match–with.

  • How the operators +, −, ∗, /, mod, +., −., ∗., and /. work on numbers.

  • How the operators =, ==, <, >, <=, >=, <>, and != work.

  • How OCaml lists [e₁; e₂ ...; eₙ] and their types t list work.

  • How the functions hd and tl work on lists.

  • How the cons constructor :: and the append operator @ work on lists.

  • How OCaml primitive data types work: bool, char, float, int, string, unit.

  • How OCaml tuples (e₁, e₂ ..., eₙ) and their types t₁ ∗ t₂ ... ∗ tₙ work.

  • How the functions fst and snd work.

  • How function types (->’s) work.

  • How unspecified type names (like 'x) work.

  • How to define new types with type.

  • How to write types with parameters (like 'key bst).

  • How to write types with multiple parameters (like ('key, 'value) alist).

  • How to define recursive types.

  • How type constructors work.

  • How to define new exceptions with exception.

  • How to raise exceptions with raise.

  • How anonymous functions (fun p₁ p₂ ... pₙ -> e) work.

  • How functions can take other functions as their arguments.

  • How to write function types with multiple ->’s.

  • What an association list is.

  • How association list functions (from the lectures) work: alGet, alPut.

  • How to represent logical propositions in OCaml (from lectures).

  • What a generator is.

  • How to write generators that take functions as arguments.

  • How the OCaml semicolon (;) operator works.

  • How the tautology checker functions (from lectures) work: evaluate, generateBools, generatePairs, generateAndTestPairs, and isTautology.

  • What a function closure is.

  • What an environment is.

  • The parts of a function closure: environment, parameters, body.

  • How functions close themselves in their defining environments.

  • What a dispatcher is.

  • What a continuation is.

  • What continuation passing style (CPS) is.

  • How to write functions in continuation-passing style.

You may also need to know about data structures and general computer science topics from prerequisite courses, such as: asymptotic analysis (‘‘big O’’), sorting, rational numbers, stacks, queues, binary trees, binary search trees, linked lists, etc.
