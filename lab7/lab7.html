0. Introduction.

This laboratory assignment asks you to write a few simple functions using OCaml’s built-in lazy evaluation mechanism.

1. Implementation.

OCaml provides an object (ungrammatically) called a lazy that implements lazy evaluation. The following module, type, operator, and functions work with lazies.

Lazy

The module Lazy contains OCaml code that implements lazies. To use lazies, you must place this line:

open Lazy ;;

early in your source file. Note that OCaml module names begin with upper case letters.

s Lazy.t

The type of a lazy that provides a value of type s. For example, a lazy that provides an integer has the type int Lazy.t, and a lazy that provides a string has the type string Lazy.t, etc. By convention, if a module M defines a type, then that type is called M.t. (Mysteriously, the type s Lazy.t can also be written as s lazy_t.)

lazy e

Return a new lazy that contains e, so it can be evaluated later. Do not evaluate the expression e yet. Note that lazy is a built-in operator in OCaml.

force l

Here l is a lazy. If force has never been called on l, then l contains an unevaluated expression. Evaluate that expression, store the expression’s value in l, and return that value. If force has been called on l before, then return the value stored in l.

You must write a type and four functions, as described below.

s lazyList

This is the type of a lazy list whose elements have the type s. Both the head and the tail of such a list must be evaluated lazily. The type must have two constructors. The constructor LazyEmpty must return an empty lazy list. The constructor LazyNode (h, t) must return a new lazy list whose head is h and whose tail is t. Both h and t must be lazies.

LazyListError

An exception. This will be raised if lazyHead, lazyTail, or lazyTake are given arguments different from what they expect.

lazyCons h t

Return a lazy list whose head is h and whose tail is t. Both h and t must be lazies.

lazyHead l

Here l is a lazy list, of type s lazyList. If l is an empty lazy list, then raise LazyListError. Otherwise return the evaluated head of l, whose type is s.

lazyTail l

Here l is a lazy list, of type s lazyList. If l is an empty lazy list, then raise LazyListError. Otherwise return the evaluated tail of l, whose type is s lazyList.

lazyTake l n

Here l is a lazy list, of type s lazyList. Return an ordinary list of the first n elements from l, where n is an integer greater than or equal to 0. The returned list must have the type s list. If l does not have at least n elements, then raise LazyListError. You may use lazyCons, lazyHead, and lazyTail as helpers.

The function lazyInts is used to help test your code. It returns a new lazy list of unevaluated integers from start to stop, inclusive. Each time an element of the lazy list is evaluated, it prints a message.

let lazyInts start stop = 
  let rec lazyInting which 
    if which <= stop 
    then lazyCons 
           (lazy (Printf.printf "Computed integer %i\n" which ; which)) 
           (lazyInting (which + 1)) 
    else LazyEmpty 
  in lazyInting start ;;

The function lazyFibs is also used to help test your code. It returns a new lazy list of all unevaluated Fibonacci numbers. The lazy list is infinitely long, ignoring arithmetic overflow. Each time an element of the lazy list is evaluated, it prints a message.

let lazyFibs () = 
  let rec lazyFibbing left right = 
    lazyCons 
      (lazy (Printf.printf "Computed Fibonacci %i\n" left ; left)) 
      (lazy (lazyFibbing right (left + right))) 
  in lazyFibbing 0 1 ;;

Note that lazyFibs would not terminate if it was evaluated eagerly!
