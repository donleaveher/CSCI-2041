0. Introduction.

A large program may be hundreds or thousands of pages long. It may use thousands of names. As a result, many programming languages provide mechanisms for dividing names into smaller and more manageable groups. OCaml does this using modules and module types. In this laboratory assignment, you will write a module that defines association lists, and a module type that describes that module.

1. Theory.

An association list is an ordered finite sequence of zero or more key-value pairs. (You’ve seen several slightly different implementations of association lists in the lectures.) If the subscripted k’s are keys, and the subscripted v’s are values, then an association list might be written (in a mathematical notation, not OCaml!) like this.

[ k1 → v1, k2 → v2 ..., kn → vn ]

We say that each key kⱼ is associated with its value vⱼ. For example, the association list A₀ associates Arabic digits with Roman numerals.

A0  =   [ 1 → I, 2 → II, 3 → III, 4 → IIII, 5 → V, 6 → VI, 7 → VII, 8 → VIII, 9 → IX ]

We can get the value of a key in an association list. For example, the value of key 4 in A₀ is IIII. We can put a key-value pair into an association list by adding it to the front, producing a new association list. For example, we can put the key 4 and its value IV into A₀, to produce a new association list A₁.

A1  =   [ 4 → IV, 1 → I, 2 → II, 3 → III, 4 → IIII, 5 → V, 6 → VI, 7 → VII, 8 → VIII, 9 → IX ]

Now the value of key 4 in A₁ is IV instead of IIII as it was before. When we get the value of a key, we search the association list from left to right for the first pair that gives a value to the key. Finally, we can also delete a key-value pair from an association list. If we delete the pair with the key 5, we get another new association list, A₂.

A2  =   [ 4 → IV, 1 → I, 2 → II, 3 → III, 4 → IIII, 6 → VI, 7 → VII, 8 → VIII, 9 → IX ]

When we delete a key-value pair in this way, we also search the association list from left to right, and we delete only the first key-value pair that we find. For example, if we deleted the pair with the key 4, we would get a new association list that associates 4 with IIII, just as A₀ did.
      Association lists are immutable. When we put a new key-value pair into a list, or delete an old key-value pair from a list, we always get a new list. We never change existing association lists.

2. Implementation.

For this laboratory assignment, you must write an OCaml module and module type that implements association lists, as described below.

Associaty

Module type. It must describe the contents of the module Association, including NoSuchKey, delete, get, make, put, and t as described below. It must not describe error, so that function will not be visible outside the module Association.

Association

Module. It must define NoSuchKey, delete, error, get, make, put, and t, as described below.

t

The type of an association list. By OCaml convention, if a module defines a data type, then that type is called t. Here the type t must have a constructor Empty that represents an empty association list. It must also have a constructor Pair that makes an ordered triple which contains a key, the key’s value, and the rest of the association list.

NoSuchKey

Exception. Raised by the function error.

delete pairs key

Function. Here pairs is an association list (of type t) and key is a key. Return a new association list that is like pairs except that the first appearance of key and its value are deleted. If key does not appear at least once in pairs then call error. Hint: Do not write delete as a tail recursion.

error ()

Function. Raise the exception NoSuchKey.

get pairs key

Function. Here pairs is an association list (of type t) and key is a key. Return the value of key in pairs. If key does not appear at least once in pairs then call error. Hint: Write get as a tail recursion.

make ()

Function. By OCaml convention, if a module defines a data type, then it defines a function called make that returns an instance of that type. Return a new empty association list (of type t).

put key value pairs

Function. Return a new association list (of type t) that is like pairs except that in this list, the value of key is value. Hint: Write one short line of OCaml code.

As you may have guessed, the point of this assignment is not to introduce association lists, since you’ve seen those before. The point is to make you think about modules and their types.
