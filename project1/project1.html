0. Introduction.

It’s possible to solve an equation numerically, by computing numbers for its variables. It’s also possible so solve an equation symbolically, by using algebra. For example, to solve the equation a × u + b = v symbolically for u, we’d first subtract b from both sides, giving a × u = v − b. Then we’d divide both sides by a, giving u = (v − b) / a. Assume that we never divide by 0. In this project, you will write an OCaml program that uses algebraic identities to solve equations. To simplify the problem, the equations will use only variables, the unary ‘−’ operator, and the binary ‘+’, ‘−’, ‘×’, and ‘/’ operators. Also, the program need only solve for a variable that appears exactly once in the equation.

1. Theory.

Here’s a mathematical description of how your program must work. Suppose that the rule L ⇒ R means that an equation L may be algebraically transformed to an equivalent equation R. For example, A + B = C ⇒ B = C − A.
      A variable is said to be inside an expression if it appears in that expression at least once. For example, the variable u is inside the expression a × u, but it is not inside the expression v − b. Each variable is inside itself, so that u is inside u.
      Now suppose that A, B, and C are expressions, and that some variable v always appears exactly once inside A, or exactly once inside B, but not both. Then the following rules show how an equation can be solved for v. Each rule always leaves v on the left side of the equation. Each rule also leaves the left side of the equation simpler than it was before the rule was applied. As a result, if we repeatedly apply the rules to an equation, one at a time, then v will eventually be left alone on the left side—and the equation is solved for v.

For example, suppose we want to solve the equation a × u + b = v for u. We first apply the ‘+’ rule, giving a × u = v − b. We then apply the ‘×’ rule, giving u = (v − b) / a. Since u is now alone on the left side, the equation is solved.

2. Implementation.

To simplify grading, you must use the OCaml type expression to represent equations and expressions. OCaml code for this type appears in the file expression.ml. The constructors have short snappy names because you’ll use them a lot. A comment after each constructor describes what it represents.

type expression = 
  Var of string |                    (∗ a, b, c ⋯ ∗) 
  Neg of expression |                (∗   − R ∗) 
  Add of expression ∗ expression |   (∗ L + R ∗) 
  Div of expression ∗ expression |   (∗ L / R ∗) 
  Equ of expression ∗ expression |   (∗ L = R ∗) 
  Mul of expression ∗ expression |   (∗ L × R ∗) 
  Sub of expression ∗ expression ;;  (∗ L − R ∗)

For example, the equation a × u + b = v is represented as an instance of expression like this.

(Equ (Add (Mul (Var "a", Var "u"), Var "b"), Var "v"))

You must write the following exception and functions. Again to simplify grading, you must use the same names for functions and their parameters that are shown here. You may also write as many additional helper functions as you need, either external helpers, internal helpers, or a mixture of the two.

SolvingError

(5 points.) Exception. It’s raised when something erroneous or unexpected happens in the functions described below. It’s helpful if SolvingError contains a string that explains why it was raised.

isInside name expression

(10 points.) A function of type string −> expression −> bool. Return true if a Var containing name appears at least once within expression. Return false otherwise.

solver name left right

(25 points.) A function of type string −> expression −> expression −> expression. This does all the work for the equation solver. Assume that left is the left side of the equation to be solved, and right is the right side. Also assume that a Var containing name is inside left. Repeatedly apply the algebraic rules shown above to left and right, producing new left’s and right’s, always leaving the Var containing name inside left. Continue until finally a Var containing name is alone as left. Then return Equ (left, right), representing the solved equation.

solve name equation

(10 points.) A function of type string −> expression −> expression. Here equation must have the form Equ (left, right). Assume that a Var containing name appears in either left or right, but not both. Call solver with left and right, so that a Var containing name is inside left. Finally return solver’s result, a solved equation.

Hint: the OCaml pattern matching mechanism (match-with) can be used to apply rules. Tail recursions can be used to apply them repeatedly until an equation is solved.

3. Example.

Here’s an example. To run the equation solver, call solve with a string that represents a variable, and an expression that represents an equation, as described above. Suppose you call solve like this.

solve "u" (Equ (Add (Mul (Var "a", Var "u"), Var "b"), Var "v")) ;;

Then solve must return a new equation like this.

Equ (Var "u", Div (Sub (Var "v", Var "b"), Var "a"))

Of course to adequately test solver, you must call it on more examples than just this one.
